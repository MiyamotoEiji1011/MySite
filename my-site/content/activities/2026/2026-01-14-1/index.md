# 導入

これまで、ちゃんとした「自分のホームページ」を持っていませんでした。
過去に何度か自作したことはあったのですが、すべてフルスクラッチ（HTML/CSS/JSを直書き）で作っていたため、更新がとにかく面倒でした。

その結果、更新しないまま放置 → ドメインの期限切れ → サイト消滅、という悲しい流れに。

最近になって、活動の幅が広がり、作ったものや考えたことをきちんと外に出したくなってきました。
そこで改めて、「長く運用できる前提」でホームページを作り直すことにしました。

# ホームページを作るにあたり

以前うまくいかなかった一番の理由は、

* 先のことを考えずにコードを書いた
* 記事追加や修正の手間が大きかった
* 構成が場当たり的だった

という点にあります。

22歳になった今、もう一度きちんと設計から考え直そうと思いました。
「作ること」よりも「続けられること」を重視して、構成・開発環境・運用方法を最初に決めます。

今回は、以下の方針でホームページを構築していきます。

* **デザイン**: Figmaで全体像とレイアウトを先に固める
* **開発環境**: Reactを使い、コンポーネント指向で保守性を高める
* **公開**: GitHub Pagesを利用してホスティング
* **ドメイン**: お名前ドットコムで取得し、GitHub Pagesに設定
* **記事の更新**: Markdown（md）ファイルを追加するだけで記事を増やせる構成
* **デプロイ**: GitHub Actionsを使い、自動デプロイを行う

この構成にすることで、「書きたいときにすぐ書ける」「壊れにくい」「後から直しやすい」サイトを目指します。

# 今後について

この記事が公開されているということは、このサイト自体はすでに完成しています。
ただ、完成したから終わり、というよりは「どういう経緯で作ったのか」「作る過程で何を考えて、どう判断したのか」を忘れないように、メモとして残していくつもりです。

このサイトでは、

* ものづくりの記録
* プロジェクトの裏側
* 試行錯誤した過程や考え方

といった内容を、あまり気負わずにまとめていこうと思います。

完成品だけでなく、その途中にある迷いや判断も含めて残しておくことで、あとから振り返ったときにも意味のある場所にしたいです。

今後も、ものづくりやプロジェクト関連の記事を中心に更新していく予定なので、もし何か気になるものがあれば、参考程度に眺めてもらえたら嬉しいです。
