# はじめに

前回の記事では、「なぜこのサイトを作ったのか」という背景の話をしました。
今回はその続きとして、実際にこのサイトをどうやって作っていったのかを、大まかにメモしていきます。

この記事が出ている時点でサイト自体はすでに完成していますが、
作った経緯や、その途中で何を考えてどう判断したのかは、時間が経つと意外と忘れてしまいます。
なので今回は、開発ログに近い感覚でまとめています。

# Figmaを使ってみて

今回、ページデザインの設計には **Figma** を使いました。
実はFigma自体は、参加していたベンチャーや企業プロジェクトで
エンジニアの方が使っているのを横で見ていただけ、という状態でした。

「とりあえず触ってみるか」くらいの気持ちで使い始めたのが正直なところです。

最初は、フロントページをそのまま全部作れるツールだと勝手に思っていたのですが、
実際にはあくまで **デザイン設計に特化したツール** でした。
正直、使う前は「意味あるのかな？」とも思っていました。

ただ、実際に使ってみると、

* ページ全体の構成を俯瞰できる
* レスポンシブを意識した配置が考えやすい
* ヘッダーや共通パーツの設計が楽

といった点がかなり便利でした。

「Webの構造に沿ってデザインの大枠を先に決める」
この工程があるだけで、実装時の迷いが一気に減った気がします。
今後も、何か作るときはまずFigmaで整理してから始めたいと感じました。

# Reactを初めて使ってみた

次に、実装部分です。
今回のサイトでは **React + Vite** の構成を採用しました。

React自体も、実は本格的に触るのは今回が初めてでした。
これまでは、いわゆるバニラな環境でゴリ押し実装をすることが多く、

* コード量が増えがち
* 修正すると別の場所が壊れる
* どこが原因か探すのが大変

という状況になりがちでした。

Reactを使ってみて一番良いと感じたのは、
**コンポーネント単位で管理できること** です。

UIごとにパーツを分けられるので、

* 変更の影響範囲が分かりやすい
* デザインの保守性が高い
* フォルダ構成自体が設計図になる

という感覚がありました。

仕事でWebアプリを触ることもあるので、
今後は個人開発・業務どちらでもReactをベースに進めていこうと思っています。

※ Reactの立ち上げコマンドや開発用コマンドについては、
自分用メモとして別途まとめておく予定です。

# ディレクトリ構成について

今回のサイトで使っているディレクトリ構成は、以下のようになっています。

```
MySite/
├── .claude/
├── my-site/
│   ├── content/
│   │   └── activities/
│   │       └── 2026/
│   │           ├── 2026-01-12/
│   │           │   ├── image.jpg
│   │           │   └── index.md
│   │           ├── 2026-01-14-1/
│   │           │   └── index.md
│   │           └── 2026-01-14-2/
│   │               └── index.md
│   ├── public/
│   ├── src/
│   │   ├── components/
│   │   ├── pages/
│   │   ├── ui/
│   │   ├── data/
│   │   ├── lib/
│   │   └── App.tsx
│   └── vite.config.ts
└─
```

記事は `md` ファイルで管理し、
UIやページはReactコンポーネントとして分離しています。

この構成のおかげで、
「記事を書く」と「コードを書く」がちゃんと分離できていて、
かなり扱いやすくなりました。

# UIデザインについて（uiverse.io）

デザイン面では、すべてを一から作るのではなく、
**uiverse.io** をかなり参考にしています。

[https://uiverse.io/](https://uiverse.io/)

ボタンやカードなどのUIがまとまっていて、
それをベースに調整・実装する形です。

「デザインを考える」と「実装する」の間をうまく埋めてくれるサイトで、
今後もUIを少し凝りたいときには積極的に使っていこうと思っています。
